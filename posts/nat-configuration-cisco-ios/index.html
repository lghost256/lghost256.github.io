Dynamic NAT Overload (PAT)

What is it?

NAT Overload, also known as PAT (Port Address Translation) is a type of NAT configuration that allows multiple devices in a network to share a single public IP address. What makes this type of NAT “dynamic” is the fact that there is no pre-defined one-to-one mapping between a private IP address and a public IP address (that is, the router takes care of the translation based on our instructions).

Configuration

The configuration steps for PAT are as follows:

1.- Define a pool of global addresses (in global configuration mode):

! NOTE: depending on your case, this step might be optional. If you're looking to specify an interface rather than a pool of addresses, look at step #3.

! PARAMETERS
! name: the name for the pool of addresses
! start-ip: first IP in the pool
! end-ip: last IP in the pool
! netmask: the subnet mask for the subnet that <start-ip> and <end-ip> belong to.

Router(config)#ip nat pool <name> <start-ip> <end-ip> netmask <netmask>

2.- Define a standard ACL to determine which internal IP addresses will be translated:

! access-list-#: from 1-99 or 1300-1999 (extended range).
! source: a local IP address.
! source-wildcard: a wildcard mask (i.e., an "inverted subnet mask") that defines the subnet.

Router(config)#access-list <access-list-#> permit <source> [source-wildcard]

3.- Enable NAT overload by indicating both the pool and the ACL (or the ACL and the interface with the public IP address):

! access-list-#: you specify the number of the ACL you've already created.
! name: you specify the name of the pool you've already created.

Router(config)#ip nat inside source list <access-list-#> pool <name> overload

Or:

! This configuration is helpful if you want to specify an interface instead of a pool of addresses.

Router(config)#ip nat inside source list 1 interface Gi0/1 overload

4.- Specify inside interfaces:

! E.g.: interface GigabitEthernet 0/0

Router(config)#interface <type> <number>
Router(config-if)#ip nat inside

5.- Specify outside interfaces:

Router(config)#interface <type> <number>
Router(config-if)#ip nat outside

Static NAT Overload

What is it?

This configuration allows external users (on the internet) to access our local services by using a single public (inside global) IP address. What makes this type of NAT static is that there is a one-to-one mapping between the internal IP address and port and the external IP address and port.

Configuration

The configuration steps are as follows:

1.- Enable static translation by providing the inside local IP & port and inside global IP & port:

! local-ip: IP address of a device in our internal network.
! local-port: port number of a service in our internal network.
! global-ip: global IP of a service in our network
! global-port: port number of a service in our network
! tcp & udp: the protocol to translate

Router(config)#ip nat inside source static {tcp | udp} <local-ip> <local-port> <global-ip> <global-port>

2.- Specify inside interfaces:

# E.g.: interface GigabitEthernet 0/0

Router(config)#interface <type> <number>
Router(config-if)#ip nat inside

3.- Specify outside interfaces:

Router(config)#interface <type> <number>
Router(config-if)#ip nat outside

Verify the NAT Configuration

  • See the NAT table:
Router#show ip nat translations
  • Delete dynamic NAT table entries:
Router#clear ip nat translation *
  • Monitor address translation in real time:
Router#debug ip nat
