What is IPsec?

IPsec is a suite of protocols used to create secure tunnels across insecure networks (like the internet). Two routers (also called peers) are used to establish the tunnel.

IPsec also provides authentication, ensuring that only authenticated peers can connect.

Finally, any traffic carried by IPsec is encrypted to maintain confidentiality.

What Traffic Does IPsec Transport?

IPsec can be configured to transport specific types of traffic based on defined rules. Traffic that matches these rules is referred to as interesting traffic. If the traffic matches one of the rules, IPsec will use the tunnel to deliver it.

What Phases Does IPsec Go Through?

IKE Phase 1

This phase is focused on exchanging keys. The goals of Phase 1 are to:

  • Authenticate peers using a pre-shared key (PSK) or a certificate.
  • Use asymmetric encryption to securely share a symmetric key, which will be used in Phase 2.
  • At the end of this phase, an IKE Phase 1 tunnel (also called a Security Association, or SA) is established.

IKE Phase 2

  • Uses the keys established in Phase 1.
  • Agrees on encryption methods and keys for bulk data transfer.
  • At the end of this phase, an IPSec SA is established, securing data traffic.

Configuration steps

1.- Use an ACL to identify VPN traffic:

! name: a name for the ACL.
Router(config)#ip access-list extended <name>
! source: the network where the VPN traffic originates from.
! destination: the network where VPN traffic is destined to.
Router(config-ext-nacl)#permit ip <source> <source-wildcard> <destination> <destination-wildcard>

2.- Configure ISAKMP policy (phase 1):

! priority: priority of protection suite (1-10000). Determines the priority of different IKE policies, ensuring the best policy is selected.
Router(config)#crypto isakmp policy <priority>
Router(config-isakmp)#encryption {3des | aes | des}
Router(config-isakmp)#authentication pre-share
! group: define the cryptographic strength used during the key exchange process.
! The size of the modulus used directly impacts the security of the key exchange.
Router(config-isakmp)#group {1 | 2 | 5}

3.- Configure ISAKMP key:

Router(config)#crypto isakmp key <secret-key> address <peer-address>

4.- IPSec transform-set (phase 2):

! name: the name of the transform-set.
Router(config)#crypto ipsec transform-set <name> esp-aes 256 esp-sha-hmac

5.- Set up crypto map:

Router(config)#crypto map <tag> <sequence-#> ipsec-isakmp
Router(config-crypto-map)#set peer <peer-address>
! pfs: enhances security because it prevents attackers from decrypting past traffic even if they obtain the key used for the current session.
Router(config-crypto-map)#set pfs {group1 | group2 | group5}
Router(config-crypto-map)#set transform-set <transform-set-name>
Router(config-crypto-map)#match address <acl-name>

6.- Apply the crypto map:

Router(config)#interface <type> <number>
Router(config-if)#crypto map <tag>

Verification Commands

Router#show crypto isakmp sa
Router#show crypto ipsec sa

Sample Configuration (Packet Tracer)

  • Goal: configure an IPsec Tunnel Between VIL and SAN. The network topology we will use is the following:

1. Activate the securityk9 Module

This enables IPsec VPN functionalities on the PT routers (I am using a 2911):

license boot module c2900 technology-package securityk9

(Note: when configuring this in PT 8.2.2, I had to execute the above command twice before I got the prompt to accept a license agreement to activate the securityk9 module. I’m assuming that this is a bug).

2. Configure Default Route on Both Routers

VIL

ip route 0.0.0.0 0.0.0.0 20.0.0.2

SAN

ip route 0.0.0.0 0.0.0.0 20.0.0.5

3. Configure Extended Access Control Lists (ACLs)

VIL

ip access-list extended VPN_ACL
permit ip 192.168.1.0 0.0.0.255 20.0.2.0 0.0.0.255

SAN

ip access-list extended VPN_ACL
permit ip 20.0.2.0 0.0.0.255 192.168.1.0 0.0.0.255

4. Configure ISAKMP (Phase 1)

VIL

crypto isakmp policy 10
encryption aes 256
authentication pre-share
group 5

SAN

crypto isakmp policy 10
encryption aes 256
authentication pre-share
group 5

5. Configure PSK (Pre-shared Key)

VIL

crypto isakmp key cisco address 20.0.0.6

SAN

crypto isakmp key cisco address 20.0.0.1

6. Define transform-set

VIL & SAN

crypto ipsec transform-set VPN-TS esp-aes 256 esp-sha-hmac

7. Configure crypto map

VIL

crypto map VPN-CM 10 ipsec-isakmp
set peer 20.0.0.6
set pfs group5
set transform-set VPN-TS
match address VPN_ACL

SAN

crypto map VPN-CM 10 ipsec-isakmp
set peer 20.0.0.1
set pfs group5
set transform-set VPN-TS
match address VPN_ACL

8. Apply crypto map to the Interfaces

VIL

interface Serial 0/3/0
crypto map VPN-CM

SAN

interface Serial 0/3/0
crypto map VPN-CM

9. Verifying functionality

By pinging from PCV (192.168.1.10) to PCS (20.0.2.10), we can verify that there is connectivity between the two devices.
