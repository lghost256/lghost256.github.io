NAT Dinámico con Sobrecarga (PAT)

¿Qué es?

NAT Overload, también conocido como PAT (Traducción de Direcciones de Puerto), es un tipo de configuración de NAT que permite que varios dispositivos en una red compartan una única dirección IP pública. Lo que hace que este tipo de NAT sea “dinámico” es el hecho de que no hay una asignación predefinida uno a uno entre una dirección IP privada y una dirección IP pública (es decir, el enrutador se encarga de la traducción según nuestras instrucciones).

Configuración

Los pasos de configuración para PAT son los siguientes:

1.- Definir un grupo de direcciones globales (en el modo de configuración global):

! NOTA: dependiendo de tu caso, este paso puede ser opcional. Si deseas especificar una interfaz en lugar de un grupo de direcciones, consulta el paso #3.

! PARÁMETROS
! nombre: el nombre para el grupo de direcciones
! start-ip: la primera IP en el grupo
! end-ip: la última IP en el grupo
! netmask: la máscara de subred para la subred a la que pertenecen <start-ip> y <end-ip>.

Router(config)#ip nat pool <nombre> <start-ip> <end-ip> netmask <netmask>

2.- Definir una ACL estándar para determinar qué direcciones IP internas serán traducidas:

! access-list-#: de 1 a 99 o 1300 a 1999 (rango extendido).
! source: una dirección IP local.
! source-wildcard: una máscara wildcard (es decir, una "máscara de subred invertida") que define la subred.

Router(config)#access-list <access-list-#> permit <source> [source-wildcard]

3.- Habilitar NAT con sobrecarga indicando tanto el grupo como la ACL (o la ACL y la interfaz con la dirección IP pública):

! access-list-#: debes especificar el número de la ACL que ya has creado.
! nombre: debes especificar el nombre del grupo que ya has creado.

Router(config)#ip nat inside source list <access-list-#> pool <nombre> overload

O:

! Esta configuración es útil si deseas especificar una interfaz en lugar de un grupo de direcciones.

Router(config)#ip nat inside source list 1 interface Gi0/1 overload

4.- Especificar las interfaces inside:

! Ejemplo: interface GigabitEthernet 0/0

Router(config)#interface <tipo> <número>
Router(config-if)#ip nat inside

5.- Especificar las interfaces outside:

Router(config)#interface <tipo> <número>
Router(config-if)#ip nat outside

NAT Estático con Sobrecarga

¿Qué es?

Esta configuración permite que los usuarios externos (en Internet) accedan a nuestros servicios locales utilizando una única dirección IP pública (inside global). Lo que hace que este tipo de NAT sea estático es que existe una asignación uno a uno entre la dirección IP interna y el puerto, y la dirección IP externa y el puerto.

Configuración

Los pasos de configuración son los siguientes:

1.- Habilitar la traducción estática proporcionando la IP y el puerto locales (inside local) y la IP y el puerto globales (inside global):

! local-ip: dirección IP de un dispositivo en nuestra red interna.
! local-port: número de puerto de un servicio en nuestra red interna.
! global-ip: IP global de un servicio en nuestra red
! global-port: número de puerto de un servicio en nuestra red
! tcp & udp: el protocolo a traducir

Router(config)#ip nat inside source static {tcp | udp} <local-ip> <local-port> <global-ip> <global-port>

2.- Especificar las interfaces inside:

# Ejemplo: interface GigabitEthernet 0/0

Router(config)#interface <tipo> <número>
Router(config-if)#ip nat inside

3.- Especificar las interfaces outside:

Router(config)#interface <tipo> <número>
Router(config-if)#ip nat outside

Verificar la Configuración de NAT

  • Ver la tabla NAT:
Router#show ip nat translations
  • Eliminar las entradas de la tabla NAT dinámica:
Router#clear ip nat translation *
  • Monitorear la traducción de direcciones en tiempo real:
Router#debug ip nat
