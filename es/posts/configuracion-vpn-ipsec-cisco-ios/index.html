¿Qué es IPsec?

IPsec es un conjunto de protocolos utilizados para crear túneles seguros a través de redes inseguras (como Internet). Se utilizan dos enrutadores (también llamados pares) para establecer el túnel.

IPsec también proporciona autenticación, asegurando que solo los pares autenticados puedan conectarse.

Finalmente, cualquier tráfico que pase por IPsec está cifrado para mantener la confidencialidad.

¿Qué Tráfico Transporta IPsec?

IPsec puede configurarse para transportar tipos específicos de tráfico según las reglas definidas. El tráfico que coincide con estas reglas se denomina tráfico interesante. Si el tráfico coincide con una de las reglas, IPsec utilizará el túnel para entregarlo.

¿Qué Fases Tiene IPsec?

Fase 1 de IKE

Esta fase se centra en el intercambio de claves. Los objetivos de la Fase 1 son:

  • Autenticar a los pares utilizando una clave compartida previamente (PSK) o un certificado.
  • Usar cifrado asimétrico para compartir de forma segura una clave simétrica, que se utilizará en la Fase 2.
  • Al final de esta fase, se establece un túnel de Fase 1 de IKE (también llamado Asociación de Seguridad, o SA).

Fase 2 de IKE

  • Utiliza las claves establecidas en la Fase 1.
  • Acuerda los métodos de cifrado y las claves para la transferencia masiva de datos.
  • Al final de esta fase, se establece un SA de IPSec, asegurando el tráfico de datos.

Configuración

1.- Usar una ACL para identificar el tráfico VPN:

! nombre: un nombre para la ACL.
Router(config)#ip access-list extended <nombre>
! origen: la red de donde proviene el tráfico VPN.
! destino: la red a la que se destina el tráfico VPN.
Router(config-ext-nacl)#permitir ip <origen> <wildcard-origen> <destino> <wildcard-destino>

2.- Configurar la política ISAKMP (fase 1):

! prioridad: prioridad del conjunto de protección (1-10000). Determina la prioridad de diferentes políticas IKE, asegurando que se seleccione la mejor política.
Router(config)#crypto isakmp policy <prioridad>
Router(config-isakmp)#encryption {3des | aes | des}
Router(config-isakmp)#authentication pre-share
! grupo: define la fuerza criptográfica utilizada durante el proceso de intercambio de claves.
! El tamaño del módulo utilizado impacta directamente en la seguridad del intercambio de claves.
Router(config-isakmp)#group {1 | 2 | 5}

3.- Configurar la clave ISAKMP:

Router(config)#crypto isakmp key <clave-secreta> address <direccion-pareja>

4.- transform-set IPSec (fase 2):

! nombre: el nombre del conjunto de transformación.
Router(config)#crypto ipsec transform-set <nombre> esp-aes 256 esp-sha-hmac

5.- Configurar el crypto map:

Router(config)#crypto map <etiqueta> <numero-sec> ipsec-isakmp
Router(config-crypto-map)#set peer <direccion-pareja>
! pfs: mejora la seguridad porque impide que los atacantes descifren el tráfico anterior incluso si obtienen la clave utilizada para la sesión actual.
Router(config-crypto-map)#set pfs {group1 | group2 | group5}
Router(config-crypto-map)#set transform-set <nombre-transform-set>
Router(config-crypto-map)#match address <nombre-acl>

6.- Aplicar el crypto map:

Router(config)#interface <tipo> <numero>
Router(config-if)#crypto map <etiqueta>

Comandos de Verificación

Router#show crypto isakmp sa
Router#show crypto ipsec sa

Configuración de Muestra (Packet Tracer)

  • Objetivo: configurar un túnel IPsec entre VIL y SAN. La topología de red que utilizaremos es la siguiente:

1. Activar el módulo securityk9

Esto habilita las funcionalidades VPN de IPsec en los enrutadores PT (estoy utilizando un 2911):

license boot module c2900 technology-package securityk9

(Nota: al configurar esto en PT 8.2.2, tuve que ejecutar el comando anterior dos veces antes de que apareciera el mensaje para aceptar el acuerdo de licencia para activar el módulo securityk9. Supongo que esto es un error).

2. Configurar la Ruta Predeterminada en Ambos Enrutadores

VIL

ip route 0.0.0.0 0.0.0.0 20.0.0.2

SAN

ip route 0.0.0.0 0.0.0.0 20.0.0.5

3. Configurar Listas de Control de Acceso Extendidas (ACLs)

VIL

ip access-list extended VPN_ACL
permitir ip 192.168.1.0 0.0.0.255 20.0.2.0 0.0.0.255

SAN

ip access-list extended VPN_ACL
permitir ip 20.0.2.0 0.0.0.255 192.168.1.0 0.0.0.255

4. Configurar ISAKMP (Fase 1)

VIL

crypto isakmp policy 10
encryption aes 256
authentication pre-share
group 5

SAN

crypto isakmp policy 10
encryption aes 256
authentication pre-share
group 5

5. Configurar PSK (Clave Compartida Previa)

VIL

crypto isakmp key cisco address 20.0.0.6

SAN

crypto isakmp key cisco address 20.0.0.1

6. Definir transform-set

VIL y SAN

crypto ipsec transform-set VPN-TS esp-aes 256 esp-sha-hmac

7. Configurar crypto map

VIL

crypto map VPN-CM 10 ipsec-isakmp
set peer 20.0.0.6
set pfs group5
set transform-set VPN-TS
match address VPN_ACL

SAN

crypto map VPN-CM 10 ipsec-isakmp
set peer 20.0.0.1
set pfs group5
set transform-set VPN-TS
match address VPN_ACL

8. Aplicar crypto map a las Interfaces

VIL

interface Serial 0/3/0
crypto map VPN-CM

SAN

interface Serial 0/3/0
crypto map VPN-CM

9. Verificación de la funcionalidad

Al hacer ping desde PCV (192.168.1.10) a PCS (20.0.2.10), podemos verificar que existe conectividad entre los dos dispositivos.
